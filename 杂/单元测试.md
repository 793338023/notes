设计原则 SOLID：

SRP-单一职责
OCP-开闭：易与扩展，抗拒修改。
LSP-里氏替换：子类接口统一，可相互替换。
ISP-接口隔离：不依赖不需要的东西。
DIP-依赖反转：构建稳定的抽象层，单向依赖（例：A => B => C， 反例：A  => B => C => A）。
在应接不暇的需求面前，还要拆模块、重构、加单测，无疑是增加工作量，显得不切实际，《重构》这本书给了我很多指导。
